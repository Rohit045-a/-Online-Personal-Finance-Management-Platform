## ğŸ¯ Project Overview

The **Online Personal Finance Management Platform** provides three main user roles:

1. **Admin**
   - Manages users and roles (Admin, Financial Advisor, User)
   - Configures system-wide settings
   - Manages financial data security settings
2. **Financial Advisor**
   - Provides personalized financial advice based on user data
   - Interacts with users via messages & feedback
   - Tracks user progress and generates reports
3. **User**
   - Tracks daily/monthly expenses
   - Sets and manages category-wise budgets
   - Receives personalized financial advice

The application uses a **Java GUI (Swing/JavaFX)** on the front-end and **JDBC + relational database (e.g., MySQL)** in the backend.

## ğŸ§© OOP Design 

The project uses core OOP concepts extensively:

- **Classes & Inheritance**
  - `User` (base class)  
    - `Admin` extends `User`  
    - `FinancialAdvisor` extends `User`  
    - `RegularUser` extends `User`  
- **Polymorphism**
  - Common methods like `login()`, `viewDashboard()` are overridden in subclasses.
  - `FinancialAdviceService` uses polymorphism to handle different advice strategies.
- **Interfaces**
  - `ReportGenerator` (for generating financial reports)
  - `SecurityConfigurable` (for security-related config classes)
  - `DatabaseOperations<T>` (generic interface for CRUD operations)
- **Exception Handling**
  - Custom exceptions:
    - `InvalidBudgetException`
    - `InsufficientDataException`
    - `DatabaseConnectionException`
  - Try-catch-finally and user-friendly error dialogs in GUI.

## ğŸ“š Collections & Generics 

The project uses Java Collections and Generics in multiple layers:

- `List<Expense>` to store user expenses
- `Map<String, Budget>` to manage category-wise budgets
- `Map<User, List<Advice>>` to maintain advice history
- Generic DAO interface: `interface GenericDao<T>` with methods like:
  - `void save(T entity)`
  - `T findById(int id)`
  - `List<T> findAll()`
- Collections are used to:
  - Build financial reports
  - Calculate statistics
  - Generate graphs and summaries

## â± Multithreading & Synchronization 

Multithreading is used for:

- Background loading of:
  - Expense data
  - Reports and charts
- Async tasks:
  - Generating financial advice
  - Exporting reports

Example usage:
- A dedicated `ReportGeneratorThread` class for generating reports without freezing the GUI.
- `synchronized` blocks/methods used when:
  - Writing to log files
  - Updating shared collections of `Expense` & `Advice` in multi-thread scenarios

## ğŸ—ƒ Database Operations Classes

## ğŸ”Œ JDBC Database Connectivity 

**JDBC is used for:**

- Establishing connections via a dedicated class:  
  `DatabaseConnectionManager`
  - Manages:
    - DB URL
    - Username & Password
    - Connection pooling (optional)
